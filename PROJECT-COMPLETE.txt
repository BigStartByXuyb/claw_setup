PROJECT OVERVIEW
================================================================================
OpenClaw Installation Wizard is a modern Windows GUI application for automated
setup and configuration of OpenClaw, a multi-channel AI assistant gateway.

Built with: Electron + React 18 + TypeScript
Theme: Muscle Lobster (Energetic orange-red design)
Platform: Windows 10/11 (64-bit)
Lines of Code: 2500+ (including styles and comments)

KEY FEATURES
================================================================================
✓ System Validation - Windows 64-bit, Node.js ≥22.0.0
✓ Dependency Detection - node, npm, git, pnpm with version checking
✓ Smart Tool Installation - Auto-install pnpm, manual guidance for others
✓ Installation Methods - Prebuilt (5 min) or Source (15 min)
✓ Channel Configuration - 6 messaging platforms
✓ Auto Service Startup - OpenClaw gateway on localhost:18789
✓ Guide Generation - Markdown usage documentation
✓ Modern UI - Gradient backgrounds, animations, responsive design

INSTALLATION FLOW (4-STEP WIZARD)
================================================================================

STEP 1: SYSTEM CHECK
  Input:  Validate Windows platform, architecture, Node.js version
  Output: ✓ Pass or ✗ Fail with detailed requirements status
  IPC:    check-system-info → {success, platform, arch, release, nodeVersion}

STEP 2: DEPENDENCY CHECK
  Input:  Scan for node, npm, git, pnpm; check environment variables
  Output: Version info, installation paths, environment status
  IPC:    check-dependencies → {dependencies, envVars}
  Action: Auto-install pnpm or provide manual download links

STEP 3: INSTALLATION METHOD
  Input:  Choose prebuilt vs source-compile installation
  Output: Installation progress (0-100%), success/error status
  IPC:    install-openclaw → {success, output/error}
  Dir:    Optional directory selection for source compilation

STEP 4: CHANNEL CONFIG
  Input:  Select messaging channels (checkboxes)
  Output: Configuration, service startup, chat window, guide generation
  Channels: Feishu | WeChat | QQ | Telegram | Discord | Slack
  IPC:    configure-channels, start-openclaw-server, open-chat-window,
          generate-guide

IPC INTERFACE SPECIFICATION
================================================================================

1. check-system-info
   Returns: {
     success: boolean,
     platform: string (e.g., 'win32'),
     arch: string (e.g., 'x64'),
     release: string (Windows version),
     nodeVersion: string,
     requirements: {
       isWindows: { met: boolean },
       is64bit: { met: boolean }
     }
   }

2. check-dependencies
   Returns: {
     dependencies: {
       [toolName]: {
         installed: boolean,
         version?: string,
         path?: string
       }
     },
     envVars: {
       [varName]: {
         set: boolean,
         value?: string
       }
     }
   }

3. install-tool(toolName: string)
   Returns: { success: boolean, output?: string, message?: string, error?: string }

4. install-openclaw({installDir: string, method: 'prebuilt' | 'source'})
   Returns: { success: boolean, output?: string, error?: string }

5. configure-channels(channels: string[])
   Returns: { success: boolean, configPath?: string, error?: string }

6. start-openclaw-server()
   Returns: { success: boolean, error?: string }

7. open-chat-window()
   Returns: { success: boolean, error?: string }

8. generate-guide(installDir: string)
   Returns: { success: boolean, guidePath?: string, error?: string }

TROUBLESHOOTING
================================================================================

Issue: Application crashes on startup
  → Press F12 to open DevTools, check Console for errors

Issue: Dependency check fails
  → Reinstall the missing tool manually
  → Run Windows PATH environment variable check

Issue: Installation hangs
  → Check internet connection
  → Verify no antivirus blocking npm/git operations
  → Try again with administrator privileges

Issue: Service won't start on port 18789
  → Port may be in use: check netstat -an | findstr :18789
  → Run application as administrator
  → Disable firewall temporarily to test
